# 추천 프로그램 스킬

효과적인 추천/리퍼럴 프로그램을 설계합니다.

## 추천 프로그램 유형

### 1. 양면 보상 (추천인 + 피추천인)
```markdown
## 구조
추천인: 5,000원 적립금
피추천인: 첫 구매 10% 할인

## 장점
- 양쪽 모두 인센티브
- 높은 참여율
- 진입 장벽 낮춤

## 썬데이허그 예시
"친구 초대하면 둘 다 5,000원!"
```

### 2. 단면 보상 (추천인만)
```markdown
## 구조
추천인: 10,000원 적립금
피추천인: 없음 (기존 혜택만)

## 장점
- 간단한 구조
- 비용 효율적

## 적합 상황
- 제품력이 강할 때
- 신규 고객 혜택이 충분할 때
```

### 3. 단계별 보상
```markdown
## 구조
1명 추천: 3,000원
3명 추천: 10,000원 + 사은품
5명 추천: 20,000원 + VIP 등급

## 장점
- 지속적 참여 유도
- 슈퍼 추천인 육성
```

### 4. 제품 기반 보상
```markdown
## 구조
3명 추천 시 무료 제품

## 예시
"친구 3명 초대하면 밤부 블랭킷 무료!"
```

## 프로그램 설계

### 보상 금액 결정
```markdown
## 계산 공식
최대 보상 = LTV × 목표 CAC 비율

예시:
- 고객 평생 가치 (LTV): 150,000원
- 목표 CAC: LTV의 20% = 30,000원
- 추천 보상: 30,000원 이내

## 배분
- 추천인: 10,000원
- 피추천인: 10,000원
- 운영 비용: 10,000원
```

### 조건 설정
```markdown
## 피추천인 조건
- 신규 가입자만
- 첫 구매 완료 시
- 최소 구매 금액 50,000원 이상

## 추천인 조건
- 기존 구매 고객만
- 월 최대 10명
- 본인 재가입 불가
```

### 추적 시스템
```markdown
## 추적 방법
1. 고유 추천 코드 (예: INKYO2024)
2. 추천 링크 (예: sundayhug.kr/r/inkyo)
3. 이메일 초대

## 추적 포인트
- 코드/링크 공유
- 클릭
- 가입
- 구매 완료
```

## 홍보 전략

### 터치포인트
```markdown
## 구매 전
- 회원가입 페이지
- 장바구니 페이지

## 구매 후
- 주문 완료 페이지
- 배송 완료 이메일
- 리뷰 작성 후

## 상시
- 마이페이지
- 앱 푸시
- 이메일 캠페인
```

### 메시지 예시
```markdown
## 구매 완료 후
"[이름]님, 구매 감사합니다! 💛
친구에게 썬데이허그를 추천하면
둘 다 5,000원 적립금 드려요!

[내 추천 코드: INKYO2024]
[친구 초대하기]"

## 이메일
제목: [이름]님의 추천 코드로 5,000원 받으세요!
내용: ...
```

## 성과 지표

### 핵심 KPI
| 지표 | 목표 | 계산 |
|------|------|------|
| 참여율 | 10% | 추천인 수 ÷ 전체 고객 |
| 전환율 | 30% | 구매자 ÷ 피추천인 |
| 바이럴 계수 | 0.3+ | 피추천 구매자 ÷ 추천인 |
| 추천 CAC | 15,000원 | 총 보상 ÷ 신규 구매자 |

### 분석 포인트
```markdown
## 슈퍼 추천인 분석
- 상위 10% 추천인 특성
- 추천 채널 분석
- 보상 최적화

## 피추천인 분석
- 첫 구매 제품
- LTV 비교 (추천 vs 일반)
- 재구매율
```

## 썬데이허그 추천 프로그램 제안

### 프로그램 구조
```markdown
## "일요일 친구" 프로그램

### 혜택
- 추천인: 5,000원 적립금
- 피추천인: 첫 구매 15% 할인

### 조건
- 피추천인 첫 구매 30,000원 이상
- 월 최대 추천 10명
- 적립금 유효기간 90일

### 추가 보상
- 3명 추천: 밤부 손수건 세트
- 5명 추천: VIP 등급 업그레이드
- 10명 추천: 신제품 먼저 받기
```

### 메시지
```markdown
## 슬로건
"좋은 건 함께, 일요일 친구"

## 카피
"아기의 꿀잠을 선물한 썬데이허그,
친한 친구에게도 알려주세요.
추천하면 둘 다 혜택 받아요! 💛"
```
