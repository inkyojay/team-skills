# A/B 테스트 스킬

효과적인 A/B 테스트를 설계하고 분석합니다.

## 테스트 프로세스

### 1. 가설 수립
```markdown
## 가설 형식
"[변경 사항]을 하면 [목표 지표]가 [예상 변화]할 것이다.
왜냐하면 [근거]이기 때문이다."

## 예시
"CTA 버튼 색상을 파란색에서 주황색으로 변경하면
클릭률이 15% 증가할 것이다.
왜냐하면 주황색이 더 눈에 띄기 때문이다."
```

### 2. 테스트 설계
```markdown
## 필수 요소
- 테스트 대상 (변수)
- 성공 지표 (KPI)
- 샘플 크기
- 테스트 기간
- 통계적 유의수준 (보통 95%)
```

### 3. 실행
```markdown
## 체크리스트
- [ ] 트래픽 분배 확인 (50/50)
- [ ] 추적 코드 설치
- [ ] QA 테스트
- [ ] 모니터링 대시보드 설정
```

### 4. 분석
```markdown
## 분석 항목
- 통계적 유의성
- 실제 vs 예상 결과
- 세그먼트별 차이
- 부작용 확인
```

## 테스트 아이디어 (우선순위별)

### 🔴 높은 영향, 낮은 노력
1. **CTA 버튼**
   - 색상 변경
   - 텍스트 변경 ("구매하기" vs "지금 시작하기")
   - 크기/위치

2. **헤드라인**
   - 혜택 중심 vs 기능 중심
   - 질문형 vs 서술형
   - 숫자 포함 vs 미포함

3. **가격 표시**
   - 월 단위 vs 일 단위
   - 할인율 표시 방식
   - 원가 취소선

### 🟡 높은 영향, 중간 노력
4. **폼 최적화**
   - 필드 수 줄이기
   - 단계 분리
   - 자동완성

5. **신뢰 요소**
   - 리뷰 위치
   - 인증 마크
   - 보증 문구

### 🟢 중간 영향, 낮은 노력
6. **이미지**
   - 제품 vs 라이프스타일
   - 사람 포함 vs 미포함
   - 배경색

## 샘플 크기 계산

```
필요 샘플 크기 = f(현재 전환율, 예상 개선율, 통계적 유의수준)

예시:
- 현재 전환율: 3%
- 목표 개선: 20% (3% → 3.6%)
- 유의수준: 95%
- 필요 샘플: 각 그룹당 약 10,000명
```

### 테스트 기간 계산
```
테스트 기간 = 필요 샘플 크기 / 일일 트래픽

예시:
- 필요 샘플: 20,000명 (양쪽 합계)
- 일일 트래픽: 1,000명
- 테스트 기간: 20일
```

## 출력 형식

### 테스트 계획서
```markdown
# A/B 테스트 계획서

## 테스트 개요
- **테스트명**: [테스트명]
- **가설**: [가설]
- **기간**: YYYY-MM-DD ~ YYYY-MM-DD

## 변수
| 버전 | 설명 |
|------|------|
| Control (A) | 현재 버전 |
| Variant (B) | [변경 내용] |

## 성공 지표
- **주요 KPI**: 전환율
- **보조 KPI**: 클릭률, 이탈률

## 예상 결과
- **예상 개선**: +X%
- **필요 샘플**: X명
- **예상 기간**: X일

## 위험 요소
- ...
```

## 세그먼트 분석

- ...
